Architecture Overview

There is no central server for media. Audio and video stream directly peer-to-peer between Android devices using WebRTC. Firebase acts only as a lightweight signaling channel — it helps two devices exchange connection info (SDP offers + ICE candidates), then steps out of the way.

For devices on different networks (WiFi vs 4G), a TURN relay server (open-source Coturn, or a cloud-hosted service like Metered.ca) relays media when a direct P2P path is impossible due to NAT.

Firestore Security Rules

// firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Devices: readable by all auth users, writable only by owner
    match /devices/{extension} {
      allow read: if request.auth != null;
      allow write: if request.auth != null
        && request.auth.token.extension == extension;
    }

    // Calls: only caller and callee can read/write
    match /calls/{callId} {
      allow read, write: if request.auth != null
        && (resource.data.caller == request.auth.token.extension
         || resource.data.callee == request.auth.token.extension);
      allow create: if request.auth != null;
    }
  }
}

WebRTC Signaling Flow

WebRTC needs a signaling channel to exchange SDP and ICE before the P2P connection opens. Firebase Firestore is the signaling channel here — both sides listen to the same document in real time.

A

Caller: Create PeerConnection + SDP Offer

Caller creates a RTCPeerConnection with ICE servers config, creates an SDP offer (createOffer()), sets it as local description, then writes it to Firestore at calls/{callId}/offer.

B

Caller: Send FCM Push to Callee

Simultaneously, caller triggers an FCM notification to the callee's device (using their stored FCM token from Firestore). This wakes the callee app even if it's killed.

C

Callee: Accept + Create SDP Answer

Callee opens the call screen, reads the offer from Firestore, sets it as remote description, calls createAnswer(), sets it as local description, and writes the answer back to calls/{callId}/answer.

D

ICE Candidate Exchange

Both sides collect ICE candidates (generated by WebRTC) and write them to Firestore subcollections: calls/{callId}/callerCandidates and calleeCandidates. Each side listens and adds the other's candidates to their PeerConnection.

E

ICE Negotiation Completes → P2P Stream

WebRTC selects the best ICE candidate pair. Direct P2P connection is established. Audio/video streams flow directly device-to-device. Firestore is no longer involved.

WebRTC Implementation (Kotlin)

WebRtcManager.kt — Core P2P Engine

class WebRtcManager(
    private val context: Context,
    private val iceServers: List<IceServer>
) {
    private lateinit var peerConnectionFactory: PeerConnectionFactory
    private var peerConnection: PeerConnection? = null
    private var localAudioTrack: AudioTrack? = null
    private var localVideoTrack: VideoTrack? = null

    fun init() {
        // Initialize WebRTC globally (once per app)
        PeerConnectionFactory.initialize(
            PeerConnectionFactory.InitializationOptions
                .builder(context)
                .createInitializationOptions()
        )
        peerConnectionFactory = PeerConnectionFactory.builder()
            .setVideoEncoderFactory(DefaultVideoEncoderFactory(...))
            .setVideoDecoderFactory(DefaultVideoDecoderFactory(...))
            .createPeerConnectionFactory()
    }

    fun createPeerConnection(observer: PeerConnectionObserver) {
        val rtcConfig = RTCConfiguration(iceServers).apply {
            sdpSemantics = SdpSemantics.UNIFIED_PLAN
            continualGatheringPolicy =
                ContinualGatheringPolicy.GATHER_CONTINUALLY
        }
        peerConnection = peerConnectionFactory
            .createPeerConnection(rtcConfig, observer)
    }

    fun addLocalAudioTrack() {
        val audioSource = peerConnectionFactory
            .createAudioSource(MediaConstraints())
        localAudioTrack = peerConnectionFactory
            .createAudioTrack("audio0", audioSource)
        peerConnection?.addTrack(localAudioTrack!!)
    }

    fun addLocalVideoTrack(surfaceView: SurfaceViewRenderer) {
        val videoCapturer = createCameraCapturer()
        val videoSource = peerConnectionFactory
            .createVideoSource(videoCapturer.isScreencast)
        videoCapturer.initialize(
            SurfaceTextureHelper.create("CaptureThread", eglBase.eglBaseContext),
            context, videoSource.capturerObserver
        )
        videoCapturer.startCapture(1280, 720, 30)
        localVideoTrack = peerConnectionFactory
            .createVideoTrack("video0", videoSource)
        localVideoTrack!!.addSink(surfaceView)
        peerConnection?.addTrack(localVideoTrack!!)
    }

    suspend fun createOffer(): SessionDescription {
        val constraints = MediaConstraints().apply {
            mandatory.add(MediaConstraints.KeyValuePair("OfferToReceiveAudio", "true"))
            mandatory.add(MediaConstraints.KeyValuePair("OfferToReceiveVideo", "true"))
        }
        return suspendCoroutine { cont ->
            peerConnection?.createOffer(object : SdpObserver {
                override fun onCreateSuccess(sdp: SessionDescription) {
                    peerConnection?.setLocalDescription(object: SdpObserver {
                        override fun onSetSuccess() = cont.resume(sdp)
                        override fun onSetFailure(e: String?) = cont.resumeWithException(Exception(e))
                        override fun onCreateSuccess(p0: SessionDescription?) {}
                        override fun onCreateFailure(p0: String?) {}
                    }, sdp)
                }
                override fun onCreateFailure(e: String?) = cont.resumeWithException(Exception(e))
                override fun onSetSuccess() {}
                override fun onSetFailure(p0: String?) {}
            }, constraints)
        }
    }

    fun addIceCandidate(candidate: IceCandidate) {
        peerConnection?.addIceCandidate(candidate)
    }

    fun toggleMute(mute: Boolean) {
        localAudioTrack?.setEnabled(!mute)
    }

    fun toggleVideo(enabled: Boolean) {
        localVideoTrack?.setEnabled(enabled)
    }

    fun close() {
        peerConnection?.close()
        peerConnection = null
    }
}

Firebase Signaling Code

SignalingRepository.kt

class SignalingRepository(
    private val db: FirebaseFirestore
) {
    // ── CALLER SIDE ──────────────────────────────────

    suspend fun sendOffer(
        callId: String,
        callerExt: String,
        calleeExt: String,
        sdp: SessionDescription
    ) {
        db.collection("calls").document(callId).set(mapOf(
            "caller"  to callerExt,
            "callee"  to calleeExt,
            "status"  to "ringing",
            "offer"   to mapOf("type" to sdp.type.name, "sdp" to sdp.description),
            "created" to FieldValue.serverTimestamp()
        )).await()
    }

    fun listenForAnswer(callId: String, onAnswer: (SessionDescription) -> Unit) {
        db.collection("calls").document(callId)
            .addSnapshotListener { snap, _ ->
                val answer = snap?.get("answer") as? Map<*, *> ?: return@addSnapshotListener
                onAnswer(SessionDescription(
                    SessionDescription.Type.fromCanonicalForm(answer["type"] as String),
                    answer["sdp"] as String
                ))
            }
    }

    fun sendCallerCandidate(callId: String, candidate: IceCandidate) {
        db.collection("calls").document(callId)
            .collection("callerCandidates").add(mapOf(
                "sdpMid"        to candidate.sdpMid,
                "sdpMLineIndex" to candidate.sdpMLineIndex,
                "candidate"     to candidate.sdp
            ))
    }

    // ── CALLEE SIDE ──────────────────────────────────

    suspend fun sendAnswer(callId: String, sdp: SessionDescription) {
        db.collection("calls").document(callId).update(mapOf(
            "answer" to mapOf("type" to sdp.type.name, "sdp" to sdp.description),
            "status" to "active"
        )).await()
    }

    fun listenForCallerCandidates(callId: String, onCandidate: (IceCandidate) -> Unit) {
        db.collection("calls").document(callId)
            .collection("callerCandidates")
            .addSnapshotListener { snaps, _ ->
                snaps?.documentChanges
                    ?.filter { it.type == DocumentChange.Type.ADDED }
                    ?.forEach { change ->
                        val d = change.document
                        onCandidate(IceCandidate(
                            d.getString("sdpMid")!!,
                            d.getLong("sdpMLineIndex")!!.toInt(),
                            d.getString("candidate")!!
                        ))
                    }
            }
    }

    suspend fun hangUp(callId: String) {
        db.collection("calls").document(callId)
            .update("status", "ended").await()
    }
}

Incoming Call via FCM Push

The callee's app may be killed or in the background. FCM (Firebase Cloud Messaging) wakes the app and shows a full-screen incoming call notification — just like a phone call.

⚠️

Key architecture point: The caller cannot directly send FCM — that requires a server key. Use a Firebase Cloud Function as a tiny serverless trigger. It's free for low volumes and keeps the server key off the device.

Firebase Cloud Function (send-call-notification)

// functions/index.js
const functions = require('firebase-functions');
const admin    = require('firebase-admin');
admin.initializeApp();

// Triggered when a new call document is created
exports.onCallCreated = functions.firestore
    .document('calls/{callId}')
    .onCreate(async (snap, context) => {
        const call  = snap.data();
        const callee = call.callee;    // e.g. "203"
        const caller = call.caller;    // e.g. "101"

        // Get callee's FCM token from devices collection
        const deviceDoc = await admin.firestore()
            .collection('devices').doc(callee).get();
        const fcmToken = deviceDoc.data()?.fcmToken;
        if (!fcmToken) return;

        await admin.messaging().send({
            token: fcmToken,
            data: {
                type:    'INCOMING_CALL',
                callId:  context.params.callId,
                caller:  caller,
                isVideo: 'true',
            },
            android: {
                priority: 'high',
                ttl: 30000,    // 30s — drop if not delivered
            }
        });
    });

Android FCM Service

class IntercomMessagingService : FirebaseMessagingService() {

    override fun onMessageReceived(message: RemoteMessage) {
        val data = message.data
        if (data["type"] == "INCOMING_CALL") {
            showIncomingCallNotification(
                callId  = data["callId"]!!,
                caller  = data["caller"]!!,
                isVideo = data["isVideo"] == "true"
            )
        }
    }

    private fun showIncomingCallNotification(callId: String, caller: String, isVideo: Boolean) {
        val fullScreenIntent = Intent(this, IncomingCallActivity::class.java).apply {
            putExtra("callId", callId)
            putExtra("caller", caller)
            putExtra("isVideo", isVideo)
            flags = Intent.FLAG_ACTIVITY_NEW_TASK
        }
        val pendingIntent = PendingIntent.getActivity(
            this, 0, fullScreenIntent,
            PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
        )
        val notification = NotificationCompat.Builder(this, "incoming_calls")
            .setSmallIcon(R.drawable.ic_call)
            .setContentTitle("Intercom — Unit $caller")
            .setContentText(if (isVideo) "Video call incoming" else "Audio call incoming")
            .setPriority(NotificationCompat.PRIORITY_MAX)
            .setCategory(NotificationCompat.CATEGORY_CALL)
            .setFullScreenIntent(pendingIntent, true)   // Shows over lock screen
            .setAutoCancel(true)
            .build()
        NotificationManagerCompat.from(this).notify(1001, notification)
    }

    override fun onNewToken(token: String) {
        // Update FCM token in Firestore whenever it refreshes
        val ext = PrefsHelper.getExtension(this)
        FirebaseFirestore.getInstance()
            .collection("devices").document(ext)
            .update("fcmToken", token)
    }
}

NAT Traversal — STUN & TURN

Devices on WiFi vs 4G are behind different NATs and cannot reach each other directly. STUN helps discover public IPs. TURN relays media when all direct paths fail.

💡

For a housing building: Use a free STUN server (Google's stun.l.google.com:19302) plus a low-cost TURN service like metered.ca or xirsys.com — typically $0–5/month for a small building. No server to manage.

ICE Servers Configuration in Kotlin

object IceConfig {
    fun getIceServers(): List<PeerConnection.IceServer> = listOf(

        // Free Google STUN — discovers public IP/port
        PeerConnection.IceServer
            .builder("stun:stun.l.google.com:19302")
            .createIceServer(),

        PeerConnection.IceServer
            .builder("stun:stun1.l.google.com:19302")
            .createIceServer(),

        // TURN relay — fallback when P2P fails (WiFi ↔ 4G)
        // Option A: Self-hosted Coturn on a ₹500/mo VPS
        PeerConnection.IceServer
            .builder("turn:your-turn-server.com:3478")
            .setUsername("intercom")
            .setPassword("yourpassword")
            .createIceServer(),

        // Option B: Metered.ca managed TURN (free tier available)
        PeerConnection.IceServer
            .builder("turn:relay.metered.ca:80")
            .setUsername("metered_username")
            .setPassword("metered_credential")
            .createIceServer()
    )
}

Door Intercom Flow

The door device (extension 000 or X00) is an Android tablet or dedicated device mounted at the entrance. When a visitor presses the call button, it triggers a video call to a specific unit or the security console (999).

1

Visitor presses call button on door device

The door Android app shows a numeric pad or a pre-programmed "call unit" button. Visitor selects unit 203 or presses the main call button.

2

Door device initiates WebRTC video call

Same flow as a regular call. Door device (ext 000) writes SDP offer to Firestore, Cloud Function sends FCM to unit 203's Android device.

3

Resident receives full-screen video call

Resident sees live video from door camera on their phone. Can see and speak to visitor via two-way audio/video WebRTC stream.

4

Resident taps "Unlock Door"

App writes a door unlock command to Firestore: doorCommands/{doorId} with a short TTL. The door device listens to this document and triggers the relay/lock GPIO or a smart lock API.

Door Unlock via Firestore

// Resident's app — tap unlock button
fun unlockDoor(doorExtension: String) {
    db.collection("doorCommands").document(doorExtension).set(mapOf(
        "command"    to "UNLOCK",
        "issuedBy"   to currentUserExtension,
        "expiresAt"  to System.currentTimeMillis() + 5000,  // 5s TTL
        "timestamp"  to FieldValue.serverTimestamp()
    ))
}

// Door device — listening for unlock commands
db.collection("doorCommands").document(myExtension)
    .addSnapshotListener { snap, _ ->
        val command   = snap?.getString("command")
        val expiresAt = snap?.getLong("expiresAt") ?: 0
        if (command == "UNLOCK" && System.currentTimeMillis() < expiresAt) {
            triggerDoorRelay()   // GPIO or smart lock SDK call
        }
    }

Security & Permissions

🔑

Firebase Phone Auth

Each resident logs in with their phone number. Firebase Auth generates a UID. Admin links UID to extension number on first setup.

🛡️

Firestore Rules

Calls collection only readable by caller and callee. Devices writeable only by their owner. Prevents snooping on neighbours.

🔒

DTLS-SRTP Encryption

WebRTC always encrypts media with DTLS-SRTP by specification. All audio/video is end-to-end encrypted — no one can intercept the stream.

⏱️

Call TTL Cleanup

Cloud Function deletes call documents after 24h. Door unlock commands have 5-second TTL. Prevents stale data buildup.

Android Runtime Permission Request

private val requiredPermissions = arrayOf(
    Manifest.permission.RECORD_AUDIO,
    Manifest.permission.CAMERA
)

private val permissionLauncher =
    registerForActivityResult(ActivityResultContracts.RequestMultiplePermissions()) { results ->
        val allGranted = results.values.all { it }
        if (allGranted) {
            startCall()
        } else {
            showPermissionRationale()
        }
    }

fun checkAndRequestPermissions() {
    val missing = requiredPermissions.filter {
        checkSelfPermission(it) != PackageManager.PERMISSION_GRANTED
    }
    if (missing.isEmpty()) startCall()
    else permissionLauncher.launch(missing.toTypedArray())
}
